BreezoMeter Weather API V1 Documentation

This document is intended for website and mobile developers who want to use *Weather data alongside Air Quality data provided by one of the BreezoMeter APIs.

The page include the following parts:

  • Current Conditions Weather API requests
  • Weather Forecast API Request (Hourly and Daily)
  • Response Data Fields
  • Icon Codes and Weather Texts Index

A proper BreezoMeter Weather API request must be of the following form:

  • https://api.breezometer.com/v1/weather/currentconditions/?parameters
  • https://api.breezometer.com/v1/weather/hourlyforecast/?parameters
  • https://api.breezometer.com/v1/weather/dailyforecast/?parameters

To access the BreezoMeter API over HTTP, use:

  • http://api.breezometer.com/v1/weather/currentconditions/?parameters
  • http://api.breezometer.com/v1/weather/hourlyforecast/?parameters
  • http://api.breezometer.com/v1/weather/dailyforecast/?parameters

While making an API request some parameters are required while some are optional. As is standard in URLs, parameters are separated using the ampersand (&) character.
*BreezoMeter Weather data is powered by Meteoblue services.

Current Conditions Weather API requests:

Current Conditions provide an estimate of the current weather conditions. Get BreezoMeter Current Conditions Weather data by Latitude and Longitude, (Geocoding), simply by concatenating the following API request:

https://api.breezometer.com/v1/weather/currentconditions/?lat={latitude}&lon={longitude}&key=YOUR_API_KEY

Required parameters:

  • lat – Latitude is a geographic coordinate that specifies the north-south position of a point on the Earth’s surface. Range between -90 to 90 (WGS84 standard)
  • lon – Longitude is a geographic coordinate that specifies the east-west position of a point on the Earth’s surface. Range between -180 to 180 (WGS84 standard)
  • key – Your BreezoMeter API key. This key identifies your account for purposes of quota management and features access.

Optional:

  • units – Response units. The default response, will be according to the Metric system, but we support also the Imperial system, which can be returned by adding “&units=Imperial”.

Responses:

The responses are returned in JSON format.

For example, the BreezoMeter Current Conditions Weather data by Latitude and Longitude response for the query on “Latitude=40.7324296, Longitude=-73.9977264”.
This request demonstrates: https://api.breezometer.com/v1/weather/currentconditions/?lat=40.7324296&lon=73.9977264&key=YOUR_API_KEY

Weather Forecast API Request

BreezoMeter Weather Forecast API requests are straightforward and simple to use.

Forecast API calls can provide:

  • Hourly forecasts weather data for a specific location for the next 24 hours, and up-to 120 hours.

  • Daily forecasts weather data for a specific location at the current or coming day, and up-to 4 next days

    Notice that at some point at the evening, (the exact time varies with the different time-zones), the current day forecast won’t be relevant anymore, hence the first object returned in the daily forecast, would be of the next day.

We use the date and time format ISO 8601 standard for our timestamp. The standard is represented as:

YYYY-MM-DDTHH:mm:SS

Forecast API queries are priced as API calls, returning 24 objects per request and priced individually per object depending on the number of hours queried, which affects the number of data objects returned. Each returned data object (maximum one per hour) is priced as one API call. The same logic applies to daily forecast, with maximum one object returned per day.

e.g The maximum price for each forecast API query will be as follows:

In Hourly Forecast API call - 5 days * 24 hour * API call price.

In Daily Forecast API call - 5 days * API call price.

Weather Hourly Forecast Data for the next 24-120 Hours:

Get BreezoMeter Hourly Forecast Weather data for the next 24 hours in a specific location by Latitude and Longitude, (Geocoding), simply by concatenating the following API request:

https://api.breezometer.com/v1/weather/hourlyforecast/?lat={latitude}&lon={longitude}&key=YOUR_API_KEY

Required parameters:

  • lat – Latitude is a geographic coordinate that specifies the north-south position of a point on the Earth’s surface. Range between -90 to 90 (WGS84 standard)
  • lon – Longitude is a geographic coordinate that specifies the east-west position of a point on the Earth’s surface. Range between -180 to 180 (WGS84 standard)
  • key – Your BreezoMeter API key. This key identifies your account for purposes of quota management and features access.

Optional:

  • units – Response units. The default response, will be according to the Metric system, but we support also the Imperial system, which can be returned by adding “&units=Imperial”.
  • hours - Hours represents the number of hours for the Hourly Forecast data you’ll receive. You can choose any value (integer) between 1 to 120 hours. 24 hours is the default period of time if you don’t use the ‘hours’ parameter.

Responses:

The responses are returned in JSON format.

For example, the BreezoMeter Hourly Forecast Weather Data for the next 48 Hours response for the query on “Latitude=40.7324296, Longitude=-73.9977264” and “hours=48”.
This request demonstrates: https://api.breezometer.com/v1/weather/hourlyforecast/?lat=40.7324296&lon=73.9977264&hours=48&key=YOUR_API_KEY

Weather Daily Forecast Data for the next 1-5 Days:

Get BreezoMeter Daily Forecast Weather data for the current/coming day, (and up-to 4 next days) in a specific location by Latitude and Longitude, (Geocoding), simply by concatenating the following API request:

https://api.breezometer.com/v1/weather/dailyforecast/?lat={latitude}&lon={longitude}&key=YOUR_API_KEY

Required parameters:

  • lat – Latitude is a geographic coordinate that specifies the north-south position of a point on the Earth’s surface. Range between -90 to 90 (WGS84 standard)
  • lon – Longitude is a geographic coordinate that specifies the east-west position of a point on the Earth’s surface. Range between -180 to 180 (WGS84 standard)
  • key – Your BreezoMeter API key. This key identifies your account for purposes of quota management and features access.

Optional:

  • units – Response units. The default response, will be according to the Metric system, but we support also the Imperial system, which can be returned by adding “&units=Imperial”.
  • days - Days represents the number of days for the Daily Forecast data you’ll receive. You can choose any value (integer) between 1 to 5 days.
    1 day is the default period of time if you don’t use the ‘days’ parameter.

Responses:

This request demonstrates:

The responses are returned in JSON format.

For example, the BreezoMeter Daily Forecast Weather Data for the next 3 Days (queried on {{ vm.displayDatetime() }} ) response for the query on “Latitude=40.7324296, Longitude=-73.9977264” and “days=3”. This request demonstrates:

https://api.breezometer.com/v1/weather/dailyforecast/?lat=40.7324296&lon=-73.9977264&days=3&key=YOUR_API_KEY

Response Data Fields:

This section explains in detail the response data fields for BreezoMeter’s global Weather data API. For Response Codes & Errors visit this page.

All measurements will be returned in the Metric system by default, but it can be customized and returned in the Imperial system as well. In case a specific weather measurement couldn't be calculated, that field will return ”null” value.

datetime
The timestamp represent the relevant, and most recent time of weather data calculations. In our API the timestamp is of ISO 8601 standard (UTC time zone).
is_day_time
Boolean value, indicating if the provided datetime is part of the day/night time. (When is_day_time is “true”, it means it is a daylight time.)
is_day_time can be used to define whether to show day or night icon image.
icon_code
Icon code is an integer value, between 1-35, which represent the relation between a weather text, and icon image.
Use icon_code together with the is_day_time field to display the correct icon image, that best describe the weather conditions and the time of the day.
weather_text
A string of weather condition interpretation for the icon_code returned.
temperature
Temperature for indicated time, at 2m above ground, provided in C/F.
feels_like_temperature
Apparent temperature for indicated time, at 2m above ground, provided in C/F.
relative_humidity
Relative Humidity is a percent (0-100) representing the air humidity.
precipitation
Sequence of information about the precipitation conditions at the indicated time.
  • precipitation_probability – Percent (0-100) representing the probability of precipitation.
  • total_precipitation - The total amount of precipitation (liquid water equivalent) that has fallen/will fall in the past/coming hour, provided in mm/in.
wind
Sequence of information about the wind conditions at the indicated time.
  • speed – The wind speed at 10m above ground, provided in km/h or mi/h.
  • direction – The wind dominant direction (In degrees) at 10m above ground.
    • 0°/360° (North): Wind blows from the North to the South (↓)  
    • 90° (East): Wind blows from the East to the West (←)
    • 180° (South): Wind blows from the South to the North (↑)  
    • 270° (West): Wind blows from the West to the East (→)
wind_gust
The wind gust speed at 10m above ground, provided in km/h or mi/h.
pressure
Pressure represent the atmospheric pressure at the surface level, provided in mb or inHg.
visibility
The visibility distance information, provided in mb or km/mi.
dew_point
The temperature at 2m above ground, provided in C/F.
cloud_cover
Percent (0-100) representing the Cover of the sky with clouds.
start_date
The timestamp represent the beginning of the day, at the location being queried, and all daily weather data calculations relates to the coming 24 hours from that time.
In our API, the timestamp is of ISO 8601 standard (UTC time zone).
sun
Information about the Sunrise and Sunset times, that are relevant to the period of 24 hours from the beginning of the day (00:00 - 23:59) according to the local time at the location being queried.
  • sunrise_time – Sunrise time in UTC.
  • sunset_time – Sunset time in UTC.
In our API the timestamp is of ISO 8601 standard (UTC time zone).

Edge cases:

Polar day
In case when the sun stays above the horizon for more than 24 hours, the field “sunrise_time“ will return: T00:00:00 (i.e. "2017-07-04T00:00:00Z") and the field “sunset_time“ will return: T24:00:00 of the same date (i.e. "2017-07-04T24:00:00Z")
Polar night
In case when the night lasts for more than 24 hours, the values in the fields “sunrise_time“ and “sunset_time“ will be: T00:00:00 (i.e. "2017-07-04T00:00:00Z")
sunrise_time, sunset_time
In polar areas, beyond 66.5 degrees, there might be missing sunrise, followed by an actual sunset if the period of polar days ends, or a missing sunset, followed by an actual sunrise if the period of polar days starts.
In these cases, the value in the correspond field will be: “---”.
moon
Sequence of information about the moon conditions, that are relevant to the period of 24 hours from the beginning of the day (00:00 - 23:59) according to the local time at the location being queried.
  • moonrise_time – Moonrise time in UTC.
  • moonset_time – Moonset time in UTC.
  • moon_phase - A string of the Moon phase name.
  • moon_age - The moon age in days.
In our API the timestamp is of ISO 8601 standard (UTC time zone).
moon_phase
The scope of values are: "new", "waxing crescent", "first quarter", "waxing gibbous", "full", "waning gibbous", "last quarter", "waning crescent"
moonrise_time, moonset_time
In case the moonrise, or the moonset does not expected to occur within the period of 24 hours, as defined above - The value in the correspond field will be: “---”.
max_uv_index
The maximum UV index (1-16) at a specific *day, at ground level.

* Day - Relates to a 24 hours period, starting at start_date, and represents one full day (00:00 - 23:59) according to the local time at the location being queried.

Icon Codes and Weather Texts Index

The table below lists all of the optional icon codes and weather texts that can be returned from BreezoMeter’s “Current Conditions” and “Hourly Forecast” Weather endpoints.

The values are returned in the icon_code and weather_text fields, and can be used for translation purposes, allocating the required space in the UI, or for creating your own set of weather icons to display with the corresponding weather text.

Displaying icon files

You can use the values returned from the icon_code together with the is_day_time field returned from BreezoMeter’s “Current Conditions” and “Hourly Forecast” Weather endpoints, to display the correct icon image, that best describes the weather conditions and the time of day.

Ideally, you can create 70 different icons - 35 for the daytime, and another 35 for night time.
